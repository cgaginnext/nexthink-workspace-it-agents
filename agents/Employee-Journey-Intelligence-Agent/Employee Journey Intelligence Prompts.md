# Employee Journey Intelligence Prompts

## Global Employee Journey Assessment

Analyze employee transition journeys over the last 90 days.

Evaluate:
- onboarding quality
- migration success
- stabilization duration
- application readiness
- workflow continuity
- collaboration experience
- productivity recovery

Analyze transitions such as:
- Windows migrations
- device replacements
- Intune onboarding
- Microsoft 365 adoption
- VDI onboarding
- Copilot rollout
- hybrid work transitions

Identify:
- high-friction journeys
- failed onboarding experiences
- migrations requiring excessive remediation
- populations struggling with adoption
- recurring stabilization issues

Provide:
- journey success rate
- stabilization timelines
- impacted populations
- recurring blockers
- estimated productivity impact
- confidence level

Highlight:
- onboarding populations with repeated friction
- migrations creating unstable devices
- high-support transitions
- silent struggling employees
- degraded collaboration experience after changes

Conclude with:
- remediation opportunities
- Engage campaigns
- Adopt guide opportunities
- workflow improvements
- automation opportunities

# New Employee Onboarding Quality

Analyze onboarding experience for employees or devices first seen in the last 60 days.

Evaluate:
- application readiness
- login experience
- collaboration readiness
- crash frequency
- missing software dependencies
- VPN usability
- Teams readiness
- authentication stability

Measure:
- time to operational readiness
- onboarding friction frequency
- support dependency
- productivity recovery

Identify:
- onboarding blockers
- repeated provisioning failures
- unstable onboarding device models
- departments with degraded onboarding quality

Recommend:
- onboarding workflow improvements
- Adopt walkthroughs
- onboarding Engage campaigns
- proactive remediation

# Migration Stabilization Analysis

Analyze post-migration device stability and employee experience.

Examples:
- Windows upgrades
- laptop refresh
- Intune migrations
- VDI transitions
- Teams migrations

Measure:
- crash amplification after migration
- support ticket evolution
- application instability
- authentication issues
- workflow degradation
- collaboration quality changes

Compare:
- pre-migration state
- transition phase
- post-migration recovery

Identify:
- migrations causing persistent instability
- hardware models with degraded outcomes
- slow recovery populations
- failed stabilization journeys

Recommend:
- targeted remediation
- adaptive rollout strategies
- post-migration automation
- personalized support

# Silent Struggle Detection

Identify employees showing repeated signs of digital friction after onboarding or migration but without creating support tickets.

Detect:
- repeated crashes
- repeated failed logins
- application abandonment
- degraded collaboration quality
- high remediation recurrence
- repeated authentication issues

Provide:
- impacted populations
- journey type
- estimated frustration severity
- likely operational blockers

Recommend:
- proactive outreach
- Engage surveys
- contextual Adopt guidance
- support prioritization
