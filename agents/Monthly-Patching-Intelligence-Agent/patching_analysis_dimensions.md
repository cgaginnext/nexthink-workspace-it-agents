# Patching Analysis Dimensions

## 1. Deployment Coverage

Analyze:

- Devices targeted
- Devices reached
- Devices patched
- Devices pending
- Devices excluded

Break down by:

- Country
- Entity
- Department
- Device model
- OS version

---

## 2. Installation Success

Measure:

- Success rate
- Failure rate
- Rollback rate
- Reboot pending rate

Analyze trends over time.

---

## 3. Build Compliance

Validate:

- KB installed
- OS build updated
- Expected revision detected

Identify devices where:
- KB present but build not updated
- Build updated but KB not reported
- Reporting inconsistencies exist

---

## 4. User Impact

Compare before and after installation:

- DEX score
- Crash rate
- Freeze rate
- Login duration
- Application responsiveness
- Collaboration quality

---

## 5. Risk Analysis

Identify:

- Repeated installation failures
- Low disk space
- Historical patching instability
- Outdated BIOS
- Driver incompatibilities
- High-risk hardware models

---

## 6. Deployment Velocity

Track:

- Time to 25%
- Time to 50%
- Time to 90%
- Time to 95%

Compare against previous monthly updates.

---

## 7. Compliance & Security

Measure:

- Devices missing latest KB
- Exposure windows
- Compliance by population
- Security posture improvement

---

## 8. Service Quality

Identify:

- Post-patch degradations
- New crash patterns
- New freeze patterns
- Increased ticket drivers
