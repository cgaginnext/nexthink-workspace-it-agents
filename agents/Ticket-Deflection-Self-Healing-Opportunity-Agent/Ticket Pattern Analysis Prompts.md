# Ticket Pattern Analysis Prompts

## Prompt — Repetitive Incident Detection

Analyze endpoint telemetry, incidents, and remote action execution history over the last 90 days to identify repetitive and avoidable support issues.

Identify:
- most frequent recurring incidents
- most impacted populations
- devices repeatedly affected
- repetitive remediation actions
- recurring manual interventions

Group incidents by:
- incident category
- application
- root cause type
- business entity
- device type
- OS version

Highlight:
- recurring support patterns
- incidents suitable for self-healing
- repetitive cache-related issues
- repeated service failures
- persistent connectivity problems

Provide:
- incident frequency
- impacted users
- estimated IT effort
- estimated employee productivity impact
- recommended automation opportunities

Prioritize:
- high-volume
- low-complexity
- high-repeatability incidents
