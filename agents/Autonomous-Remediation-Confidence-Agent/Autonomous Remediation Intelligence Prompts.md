# Autonomous Remediation Intelligence Prompts

## Global Autonomous Remediation Assessment

Analyze remote action executions and self-healing activity over the last 90 days to identify remediation candidates suitable for autonomous execution.

Evaluate:
- remediation success rate
- rollback frequency
- recurrence after remediation
- retry frequency
- employee impact
- execution consistency
- targeting precision
- remediation duration
- post-remediation stability

Identify:
- highly reliable remediations
- unstable remediations
- repetitive manual fixes
- low-risk automation candidates
- environment-sensitive remediations

Group findings by:
- remote action
- remediation category
- business entity
- device type
- OS version
- hardware model
- trigger type

Provide:
- confidence score
- automation maturity level
- estimated operational gain
- employee disruption risk
- rollback risk
- implementation recommendation

Highlight:
- remediations suitable for trusted autonomous execution
- scripts requiring stabilization
- remediations causing inconsistent results
- repetitive technician interventions
- quick-win automation opportunities

Conclude with:
- progressive automation roadmap
- governance recommendations
- automation expansion opportunities
- risk mitigation strategies

# Remediation Reliability Analysis

Analyze Remote Actions with more than 500 executions over the last 90 days.

For each remediation calculate:
- execution success rate
- failure recurrence rate
- average execution duration
- rollback frequency
- user disruption indicators
- post-remediation DEX improvement
- repeated execution necessity

Categorize remediations into:
- highly predictable
- moderately reliable
- environment-sensitive
- operationally risky

Identify:
- remediations consistently improving DEX
- remediations requiring frequent retries
- remediations with unstable outcomes
- scripts strongly impacted by hardware or OS variance

Provide:
- confidence level
- autonomous execution recommendation
- targeting restrictions
- suggested rollout model

# Recurrence Elimination Analysis

Analyze whether remediations eliminate issues durably or only temporarily.

Compare:
- incident recurrence before remediation
- recurrence after remediation
- repeated execution necessity
- subsequent ticket creation
- repeat support dependency

Identify:
- durable remediations
- temporary symptom mitigation
- unstable remediations
- false-positive remediation success

Provide:
- remediation durability score
- automation suitability
- confidence level
- operational recommendation
