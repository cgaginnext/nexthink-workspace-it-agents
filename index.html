What is a Nexthink IT Agent? Think of an IT Agent as a specialized extension of your own IT team — a highly focused virtual team member dedicated to a specific Workplace challenge, not a generic chatbot.

Usage modes

Workspace IT Agents work in two complementary modes — interactive and scheduled — enabling both reactive investigation and proactive operations.

Interactive Discussions

Launch agents directly from the Nexthink Workspace agent menu. A dedicated discussion is created with the agent as active context.

  • Investigations
  • Troubleshooting
  • Exploratory analysis
  • Root cause analysis
  • Ad hoc reporting

Scheduled Tasks

Run agents automatically on a recurring schedule without manual interaction. The agent acts as if an operator launched the prompt at the scheduled moment.

  • Daily DEX reviews
  • Security posture monitoring
  • Executive reporting
  • Automation governance
  • Proactive detection

Recommended operational model

Usage mode Best for
Interactive Discussions Investigations, troubleshooting, exploratory analysis
Scheduled Tasks Proactive monitoring, recurring reporting, continuous operational intelligence

Together, both modes enable reactive investigations, proactive detection, scalable AI-assisted operations, and continuous DEX intelligence.

How to launch an interactive discussion

1. Open the Nexthink Workspace agent menu
2. Select the desired IT Agent
3. A dedicated discussion is automatically created
4. The agent becomes the active operational context
5. Start asking questions — the conversation inherits the agent's role, expertise, and operational guidance

Scheduled tasks

Tasks transform agents from reactive assistants into proactive operational analysts running continuously without manual interaction.

Technical Preview: Tasks are currently a Technical Preview capability in Nexthink Workspace and may evolve over time.

How tasks work

1. Select an IT Agent
2. Write a prompt or operational instruction
3. Configure a schedule (daily, hourly, weekly, one-time)
4. Let Nexthink Assist execute the analysis automatically

Why tasks are powerful

Earlier issue detection

Surface emerging risks before they impact employees — no manual trigger required.

Reduced operational noise

Continuously analyze and prioritize, delivering only actionable signals.

Continuous DEX visibility

Proactive monitoring of performance, security posture, and collaboration quality around the clock.

Scalable AI operations

Industrialize proactive IT operations without growing the team headcount.

How to create a task

1. Open the Tasks view in Nexthink Assist
2. Click + New Task
3. Select an IT Agent
4. Write your prompt or instruction
5. Configure the schedule
6. Save and activate the task

Full example — PC Slowness Analyst (daily task)

PC Slowness Analyst · Daily at 9:00 AM
Identify the top 10 workplace devices showing recurring PC slowness over the last 30 days, excluding servers and focusing only on devices matching the defined naming pattern. Prioritize devices with user-visible degradation, repeated performance issues, and quick-win remediation opportunities such as low disk space, pending reboot, excessive reboot age, Windows Update backlog, high CPU Queue Length, memory pressure, OneDrive synchronization backlog, or abnormal endpoint security and management tool activity. For each device, explain the main symptoms, likely causes, recurrence pattern, recommended action, expected benefit, operational effort, and follow-up indicator. Clearly distinguish recurring structural issues from isolated episodes. At the start of each analysis, include the date and time.

Full example — Workforce Sentiment Inference Agent (daily task)

Workforce Sentiment Inference Agent · Daily at 9:00 AM
Identify the 10 most structurally unstable devices with the greatest negative impact on DEX, focusing on recurring and systemic degradation rather than isolated incidents. Correlate endpoint and application degradations, identify the most impactful root causes, and propose high-impact remediation recommendations prioritized by operational impact. At the start of each analysis, include the date and time.

Full example — Value Realization Agent (weekly task) -- For more examples See agent readme

Weekly Value Realization Review · Weekly at 9:00 AM every Monday
Produce a weekly Value Realization Review using an ultra-conservative methodology. Calculate the measurable value generated during the last 7 days from: - ticket avoidance, - Engage campaigns, - Remote Actions, - Nexthink Flows, - automated remediations, - investigation time reduction, - license optimization, - hardware lifecycle extension, - compliance improvements, - proactive incident prevention, - productivity restoration, - DEX improvements. For each value stream: - describe the activity observed, - quantify the population impacted, - explain the methodology used, - clearly separate measured value from estimated value, - use the lowest reasonable assumptions whenever uncertainty exists. Then identify the remaining unrealized opportunities. Estimate the additional value that could be unlocked by: - activating unused Nexthink library capabilities, - expanding successful automations, - improving Remote Action coverage, - reclaiming unused licenses, - extending device lifecycle, - addressing recurring DEX degradations, - increasing self-healing adoption, - improving campaign targeting. Rank all opportunities by: 1. potential value, 2. implementation effort, 3. expected time to realization. Provide: - Weekly Value Generated (€) - Weekly IT Hours Saved - Weekly Productivity Hours Restored - Weekly Tickets Avoided - Remaining Addressable Value (€) - Top 10 Value Opportunities - Quick Wins (≤30 days) - Strategic Opportunities (>30 days) Finish with an executive summary suitable for a CIO or DEX Steering Committee, explaining: - value realized this week, - value still available, - recommended priorities for the next 30 days. Use concise, evidence-based, and highly conservative assumptions throughout.
